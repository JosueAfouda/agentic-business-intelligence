# GEMINI.md - Agentic Business Intelligence Workflow

This project implements an AI-driven Business Intelligence (BI) workflow using the Gemini CLI to transform natural language questions into SQL, execute them against a PostgreSQL database, and generate interactive visualizations.

## Project Overview
- **Purpose**: Autonomous, agent-centric BI for extracting insights from relational databases without manual SQL writing.
- **Architecture**:
    - **Intention**: Natural language questions stored in `requests/*.txt`.
    - **Context**: Database schema documentation generated in `schema/`.
    - **Logic**: SQL queries generated by Gemini CLI in `sql/`.
    - **Execution**: Data extraction to CSV in `outputs/`.
    - **Visualization**: Plotly-based HTML reports generated in `outputs/` via `dataviz/` scripts.
- **Tech Stack**: Python 3, PostgreSQL, Gemini CLI, Pandas, Plotly Express, `psycopg2`, `python-dotenv`.

## Building and Running

### Setup
1. **Environment**: Ensure a `.env` file exists with the following variables:
   ```env
   DB_HOST=localhost
   DB_PORT=5432
   DB_NAME=dvdrental
   DB_USER=your_user
   DB_PASSWORD=your_password
   ```
2. **Dependencies**:
   ```bash
   pip install -r requirements.txt
   ```

### Execution Workflow (Explicit Control)
Each script now requires an explicit file argument. Run these as modules from the project root:

1. **Update Database Schema**:
   ```bash
   python3 -m scripts.schema
   ```

2. **Generate SQL from Request**:
   ```bash
   python3 -m scripts.generate_sql --request requests/my_question.txt
   ```

3. **Execute SQL and Export CSV**:
   ```bash
   python3 -m scripts.run_analysis --sql sql/my_question.sql
   ```

4. **Generate Visualization Script**:
   ```bash
   python3 -m scripts.generate_dataviz --request requests/my_question.txt
   ```

5. **Run Visualization (Generate HTML)**:
   ```bash
   python3 -m scripts.run_dataviz --dataviz dataviz/my_question.py
   ```

### Legacy/TUI (Internal Workflow)
- **TUI Interface**: `python3 scripts/tui.py` (Note: This may require updates to match the new explicit argument logic of the underlying scripts).

## Directory Structure
- `scripts/`: Core logic and CLI tools.
- `requests/`: Input directory for business questions.
- `sql/`: Generated SQL files.
- `schema/`: Database schema context files.
- `outputs/`: Final results (CSV, Metadata, HTML).
- `dataviz/`: Generated Python scripts for rendering charts.
- `utils/`: Common utilities for database connectivity and helpers.

## Development Conventions
- **Explicit Execution**: The system favors one-to-one mapping (One command = One file processed) for better user control and safety.
- **Module-based Execution**: Always use `python3 -m scripts.<script_name>` to ensure internal package imports (like `utils`) are resolved correctly.
- **Auditability**: Every generated SQL and metadata file is preserved for review.
- **Refactoring Note**: Directory iteration logic has been removed from individual scripts in favor of explicit CLI arguments.
