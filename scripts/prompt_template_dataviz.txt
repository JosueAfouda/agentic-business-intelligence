## CHART IMPLEMENTATION MAPPING (PLOTLY - STRICT)

For each selected chart type, you MUST use the corresponding Plotly implementation below.

---

### BAR / COLUMN CHARTS

**Clustered column chart**
→ `plotly.express.bar`

* x = category
* y = value
* color = optional grouping

**Clustered bar chart**
→ `plotly.express.bar`

* orientation = 'h'

---

**Stacked column chart**
→ `plotly.express.bar`

* barmode = 'stack'

**Stacked bar chart**
→ `plotly.express.bar`

* orientation = 'h'
* barmode = 'stack'

---

**100% Stacked column chart**
→ `plotly.express.bar`

* barmode = 'relative'
* normalize values to percentage

**100% Stacked bar chart**
→ same as above with orientation='h'

---

### LINE / AREA

**Line chart**
→ `plotly.express.line`

* x = time or ordered dimension
* y = value
* color = optional grouping

---

**Area chart**
→ `plotly.express.area`

**Stacked area chart**
→ `plotly.express.area`

* multiple categories stacked

**100% Stacked area chart**
→ normalize values before plotting
→ then `px.area`

---

### COMBO CHARTS

**Line and clustered column chart**
→ `plotly.graph_objects`

* use `go.Bar` + `go.Scatter`

**Line and stacked column chart**
→ `plotly.graph_objects`

* stacked bars + line overlay

---

### SPECIALIZED CHARTS

**Scatter plot**
→ `plotly.express.scatter`

---

**Pie chart**
→ `plotly.express.pie`

**Donut chart**
→ `plotly.express.pie`

* hole=0.4

---

**Treemap**
→ `plotly.express.treemap`

---

**Waterfall chart**
→ `plotly.graph_objects.Waterfall`

---

**Funnel**
→ `plotly.express.funnel`

---

**Ribbon chart**
→ NOT natively supported in Plotly
→ approximate using:

* `plotly.express.area` with ordering
  OR
* `plotly.graph_objects` with filled lines

---

### MAPS

**Map**
→ `plotly.express.scatter_mapbox`

**Filled map**
→ `plotly.express.choropleth`

---

### KPI / SUMMARY

**Card**
→ Use `plotly.graph_objects.Indicator`

* mode = "number"

---

**Multi-row card**
→ multiple `go.Indicator` in a subplot layout

---

### GAUGE

**Gauge**
→ `plotly.graph_objects.Indicator`

* mode = "gauge+number"
